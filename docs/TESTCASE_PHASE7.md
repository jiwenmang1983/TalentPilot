## 3. E2E 完整链路测试

**从职位发布到报告生成的完整流程（小P自动化 + Mark人工结合）：**

### 流程：职位→简历→匹配→面试→报告→通知

```
Step 1: HR登录 → 发布职位（产品经理）
Step 2: 创建测试候选人 + 上传简历
Step 3: 运行简历解析（AI解析入库）
Step 4: 运行智能匹配（AI评分）
Step 5: HR查看匹配结果，决定发起面试
Step 6: 发送面试邀请（触发邮件通知）
Step 7: 候选人确认面试时间
Step 8: AI面试会话创建（生成个性化问题）
Step 9: 候选人进入面试 → AI问答
Step 10: 面试结束 → AI生成报告
Step 11: HR查看报告 → 录用/淘汰
Step 12: 录用通知发送（通知系统验证）
```

**自动化验证点（python3）：**
- 每个Step的API返回码 = 200
- 数据在各Step之间正确流转
- 数据库中有对应记录

**人工验证点（Mark截图）：**
- Step 1: 职位列表显示是否正常
- Step 4: 匹配分数/理由UI是否符合预期
- Step 6: 面试邀请页面布局
- Step 9: AI面试界面是否可用
- Step 10: 面试报告排版/内容展示

## 4. T-17 边缘失败用例重测

| 用例 | 原失败原因 | 重测方案 |
|------|-----------|---------|
| 编辑部门 | 选了根节点（不可编辑） | 选择非根节点测试编辑 |
| 删除部门 | 选了有子部门的节点 | 先删子节点再删父节点 |
| 编辑角色 | 选了预置角色（不可编辑Key） | 编辑自定义角色 |
| 删除自定义角色 | 选了预置角色 | 删除自己创建的角色 |
| 用户角色下拉 | 单选vs多选混淆 | 验证单选下拉行为 |
| 表单验证溢出 | drawer高度超出视窗 | 缩小窗口验证 |
