## 1. 项目概述

### 1.1 项目背景

TalentPilot 是一个 AI 驱动的智能招聘系统，自动化招聘全流程：需求发布 → 自动化分发到外部渠道 → 多渠道简历采集 → AI 简历匹配 → AI 视频面试 → 结构化面试报告。

### 1.2 项目目标

- 将 HR 从重复性简历筛选和初筛面试中解放出来
- 通过 AI 匹配提升候选人质量
- 通过标准化 AI 面试报告辅助用人决策

### 1.3 系统边界

- 上线范围：招聘需求发布、自动化分发、AI 简历采集与匹配、AI 异步视频面试、面试结果汇报
- 暂不在范围内：候选人管理后台、offer 管理、入职流程

---

## 2. 业务流程

```
职位发布
    ↓
自动化分发到外部渠道（招聘平台/社交平台）
    ↓
AI 多渠道采集简历
    ↓
简历去重入库
    ↓
AI 匹配（JD vs 简历）
    ↓
高匹配度候选人 → 发送面试邀请
    ↓
AI 异步视频面试
    ↓
生成结构化面试报告
    ↓
HR / 用人经理查看报告，决定后续流程
```

## 3. 功能模块

### 3.1 职位管理（招聘需求发布）

- HR 创建并发布职位 JD
- 职位字段：职位名称、部门、职责描述（JD文本）、任职要求（技能/经验/学历）
- 职位状态：草稿 / 发布中 / 暂停 / 已结束

### 3.2 自动化发布到外部渠道

- HR 选择目标渠道（猎聘/拉勾/Boss直聘/领英/小红书/自定义链接）
- AI Agent 自动将 JD 内容适配各渠道格式后发布
- 支持定时发布（发布窗口）
- 发布状态追踪（已发布/发布失败/待发布）
- 渠道账号管理（各平台 API 凭证或登录账号存储）

**渠道接入优先级（三层策略）：**
1. **官方 API** — 平台提供官方接口，优先对接
2. **第三方聚合平台** — 官方 API 不可用时，使用第三方招聘聚合平台
3. **浏览器自动化采集** — 平台既无 API 也无聚合平台时，AI Agent 模拟登录账号抓取页面（如 Boss 直聘等）

### 3.3 AI 简历采集

- **触发机制：事件驱动** — 职位发布/更新时立即触发一次全量采集；之后定时增量更新
- AI Agent 从各渠道自动采集候选人简历
- **采集方式：与渠道接入策略一致（官方 API → 聚合平台 → 浏览器自动化）**
- 简历解析：提取关键字段（姓名/电话/邮箱/工作经历/学历/技能）
- 去重：同一候选人跨渠道去重，以最新简历为准
- 数据入简历库

### 3.4 AI 简历匹配

- **匹配引擎：** 将候选人简历与职位 JD 进行多维度匹配，综合评分（0-100分）
- **技能匹配：** LLM 语义分析判断（非纯关键词命中），兼容同一技能的不同行业叫法
- **年限匹配：** LLM 语义判断（如 JD 要求"3-5年"，简历"4年"→ 符合）
- **匹配维度因子（初定7-8个）：**
  1. 技能匹配度
  2. 工作经验年限匹配度
  3. 学历匹配度
  4. 行业背景匹配度
  5. 职位级别匹配度
  6. 薪资期望匹配度（可选）
  7. 地理位置匹配度
  8. 跳槽频率匹配度（可选）
- **综合评分算法：** 各维度加权求和；权重初始按行业经验设定，后续随数据积累持续调优
- **阈值规则：** HR 可在后台设定匹配度阈值（默认 80分）；高于阈值的候选人进入"待通知"状态
- **候选人列表：** 按匹配度从高到低排序

### 3.4.1 候选人通知

- **邮件：** 始终发送，简历/平台上如有邮箱则发送面试邀请
- **短信：** 有手机号则发送，无手机号则跳过
- **通知时机：** 匹配度超过阈值后，由 HR 手动确认发送，或按规则自动发送

### 3.5 AI 实时面试

- **问题来源：混合模式**
  1. **HR 预设问题：** HR 为职位配置固定面试问题（必问，AI 按顺序提问）
  2. **AI 动态追问：** AI 根据候选人的回答，自动生成追问问题（深入挖掘）
- 系统生成带时效性的面试邀请链接，发送给候选人
- 候选人可在有效期内自主选择合适时间进入面试（窗口期相对宽松）
- **实时对话：** AI 即时提问，候选人即时回答，全程 AI 主导
- **HR 实时通知：** 候选人进入面试时，系统立即通知 HR
- **HR 实时监控：** HR 可在后台实时观看面试过程（如需要）
- 面试结束后，AI 对全程进行评估，生成结构化面试报告

### 3.6 面试报告与汇报

每场面试生成一份结构化报告，包含：

**候选人基础信息**
- 姓名、学历、工作年限、应聘职位

**JD 匹配度回顾**
- 简历匹配各维度得分（技能/年限/学历/行业等）

**面试过程记录**
- 面试总时长
- 各问题回答完整度分析

**AI 评估结果**
- 综合评分
- 各维度评分
- AI 关键点评语

**结论**
- 候选人亮点
- 潜在风险点
- 是否推荐（强烈推荐 / 推荐 / 待观察 / 不推荐）

**报告操作**
- HR / 用人经理可在线查看报告
- 支持批量导出报告（PDF/Excel）

## 4. 数据权限规则

### 4.1 角色定义

- **管理员 (Admin)** — 可查看和管理所有数据
- **HR** — 仅可查看和管理自己创建的职位相关数据（自己发布的职位、候选人、面试报告）

### 4.2 权限说明

| 角色 | 全局数据 | 自己创建的职位数据 |
| --- | --- | --- |
| 管理员 | ✅ 全部可查看/管理 | ✅ 全部可查看/管理 |
| HR | ❌ 不可查看他人数据 | ✅ 可查看/管理 |
