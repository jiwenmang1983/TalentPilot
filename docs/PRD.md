## 1. 项目概述

### 1.1 项目背景

TalentPilot 是一个 AI 驱动的智能招聘系统，自动化招聘全流程：
**需求发布 → 跨渠道分发 → 简历采集 → AI 匹配筛选 → AI 实时面试 → 结构化面试报告**

招聘是每个企业最高频的运营痛点之一：HR 每天花费大量时间在简历筛选和初筛电话上，而这些工作 80% 可以被 AI 替代。根据 HireVue 2026 AI 招聘报告，AI 驱动的招聘平台可以让筛选时间减少 60%，招聘周期从 20-30 天缩短到 24 小时（Harver 客户案例）。

当前市场上已有的解决方案（Moka、大易、北森等）主要是 ATS（招聘管理系统），偏流程管理，AI 功能多为辅助筛选。真正做到"AI Agent 全流程自动化"的产品仍然稀缺，尤其是面向中国本土招聘渠道（Boss直聘、猎聘、拉勾）集成的产品。

### 1.2 项目目标

| 目标 | 衡量指标 |
|---|---|
| 减少 HR 简历筛选工作量 | AI 预筛选覆盖 80%+ 投递量 |
| 提升候选人质量 | 高匹配候选人面试通过率 > 40% |
| 缩短招聘周期 | 从投递到报告交付 < 48 小时（不含候选人等待时间） |
| 提升候选人体验 | 面试完成率 > 85%（参考 HireVue 行业基准 95%） |
| 保障招聘公平性 | AI 模型通过第三方偏见审计 |

### 1.3 系统边界

**上线范围：**
- 招聘需求发布
- 自动化分发到外部渠道
- AI 多渠道简历采集与解析
- AI 简历多维度匹配
- AI 实时视频面试
- 结构化面试报告生成与查看

**暂不在范围内（未来迭代）：**
- 候选人管理后台（候选人自主更新简历）
- Offer 管理
- 入职流程
- 内部推荐管理
- 猎头渠道管理

## 3. 功能模块

### 3.1 职位管理（招聘需求发布）

**F-01：职位创建**

HR 创建新职位，包含以下字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| 职位名称 | 文本 | 如"高级前端工程师" |
| 所属部门 | 下拉/树形 | 从组织架构中选择 |
| 职位类型 | 枚举 | 社会招聘 / 校园招聘 / 实习 |
| 职责描述（JD文本） | 富文本 | AI 用于简历匹配的核心依据 |
| 任职要求 | 结构化 | 见下方任职要求子字段 |
| 面试问题配置 | 结构化 | HR 预设问题列表（见 3.5） |
| 目标渠道 | 多选 | 猎聘 / 拉勾 / Boss直聘 / 领英 / 小红书 / 自定义 |
| 匹配阈值 | 数字 | 默认 80，HR 可调整（0-100） |
| 薪资范围 | 范围 | 最低 ~ 最高（可选，用于薪资匹配） |
| 工作地点 | 文本 | 用于地点匹配 |
| 发布状态 | 枚举 | 草稿 / 发布中 / 暂停 / 已结束 |

**任职要求子字段：**

| 子字段 | 说明 |
|---|---|
| 技能清单 | 如 Vue/React/TypeScript（AI 语义匹配） |
| 工作年限 | 如"3-5年"（AI 语义解析） |
| 学历要求 | 枚举：不限 / 高中 / 大专 / 本科 / 硕士 / 博士 |
| 行业背景 | 如"互联网/金融/教育"等 |
| 职位级别 | 枚举：初级 / 中级 / 高级 / 专家 / 管理 |
| 优先条件 | 文本（加分项，非必须） |

**F-02：职位发布**

- HR 确认 JD 后点击"发布"，AI Agent 自动执行分发流程
- 发布前可预览各渠道适配后的内容
- 支持"定时发布"（设置发布时间窗口）
- 发布状态实时追踪：待发布 → 发布中 → 已发布 / 发布失败
- 发布失败时显示失败原因（如账号未登录/配额用尽）

**F-03：职位管理列表**

- 列表展示所有职位：状态/部门/创建人/发布时间/投递数/匹配候选人数
- 支持按状态/部门/创建人筛选
- 支持对已发布职位执行"暂停""重新发布""编辑"（编辑后需重新发布）

### 3.2 跨渠道智能分发

**F-04：渠道账号管理**

HR 在后台配置各渠道的接入凭证：

| 渠道 | 接入方式 | 说明 |
|---|---|---|
| 猎聘 | 官方 Open API | 需要 HR 提供猎聘企业 API Key |
| 拉勾 | 官方 Open API | 需要 HR 提供拉勾 API Key |
| Boss直聘 | 浏览器自动化 | AI Agent 模拟登录操作（无官方 API） |
| 领英（LinkedIn） | 官方 API / 浏览器自动化 | 取决于账号权限 |
| 小红书招聘号 | 浏览器自动化 | 企业号内容发布接口 |
| 自定义链接 | 文本 | HR 提供第三方招聘页 URL |

**接入策略优先级（按渠道）：**

1. **官方 Open API**（最优先）— 稳定、合法、可大规模操作
2. **第三方聚合平台** — 当官方 API 不可用时（如 Broadbean / Textkernel）
3. **浏览器自动化**（最后兜底）— 仅用于无 API 的平台；需处理验证码/登录态/反爬限制

**F-05：AI 内容适配**

AI Agent 将 JD 内容自动适配为各渠道要求的格式：

- **Boss直聘：** 转化 JD 为简洁版，要求 500 字以内，自动提取关键技能标签
- **猎聘：** 按猎聘 JD 模板格式填充，支持长 JD
- **拉勾：** 适配拉勾的职位亮点字段（要求企业填写"为什么选择我们"）
- **领英：** 适配领英的职位描述格式（结构化 bullet points）
- **小红书：** 将 JD 转化为社交化招聘文案（面向 C 端用户，更活泼）

**F-06：发布状态追踪**

| 状态 | 说明 |
|---|---|
| 待发布 | 任务已创建，等待执行 |
| 发布中 | AI Agent 正在执行发布 |
| 已发布 | 各渠道全部/部分成功发布 |
| 部分失败 | 部分渠道成功，部分失败（显示失败原因） |
| 发布失败 | 全部失败（需要 HR 介入检查账号） |

### 3.3 AI 简历采集与解析

**F-07：多渠道简历采集**

采集触发机制（事件驱动 + 定时任务）：

- **事件触发：** 职位发布/更新时，立即触发一次全量采集
- **定时增量：** 职位发布期间，每 6 小时增量采集一次新投递
- **手动触发：** HR 可随时点击"立即采集"

采集方式优先级与分发策略一致（官方 API → 聚合平台 → 浏览器自动化）。

**F-08：简历解析**

AI 对采集到的原始简历（Word / PDF / HTML / 文本）进行解析，提取结构化字段：

| 解析字段 | 说明 |
|---|---|
| 姓名 | 候选人姓名 |
| 手机号 | 用于短信通知 |
| 邮箱 | 用于邮件通知 |
| 当前公司 | 最近一家在职公司 |
| 职位名称 | 当前/最近职位 |
| 工作年限 | 从第一份工作到现在的总年限 |
| 工作经历 | 列表：时间范围 / 公司 / 职位 / 职责描述 |
| 学历信息 | 最高学历 / 学校 / 专业 / 毕业时间 |
| 技能清单 | 从工作经历和自我介绍中提取的技能关键词 |
| 薪资期望 | 如简历中有，提取期望薪资范围 |
| 最近更新时间 | 简历最后更新时间（用于判断是否是最新的） |

**F-09：跨渠道去重**

- 以"姓名 + 手机号"或"姓名 + 邮箱"作为去重依据
- 同一候选人多渠道投递，合并为一条简历记录
- 合并策略：取各渠道中最新的工作经历，取最完整的技能描述
- 去重后记录关联来源渠道列表

**F-10：简历库管理**

- HR 可查看所有采集到的简历
- 列表按匹配度倒序（匹配度高的在前）
- 支持按职位/来源渠道/匹配度范围筛选
- 每份简历显示：姓名/职位/匹配度/采集来源/更新时间
- HR 可手动调整候选人的匹配度阈值（override）

### 3.4 AI 简历多维度匹配

**F-11：匹配引擎**

这是 TalentPilot 的核心技术模块。参考 HireVue 的"Skills-based hiring"理念，TalentPilot 的匹配不是关键词命中，而是 LLM 语义理解。

**匹配维度（共 8 个维度）：**

| # | 维度 | 权重（默认） | 匹配逻辑 |
|---|---|---|---|
| 1 | 技能匹配度 | 30% | LLM 语义分析 JD 技能 vs 简历技能，兼容同义不同表述 |
| 2 | 工作年限匹配度 | 20% | LLM 解析 JD 年限要求 vs 简历实际年限 |
| 3 | 学历匹配度 | 15% | JD 要求学历 vs 简历最高学历 |
| 4 | 行业背景匹配度 | 10% | JD 行业 vs 简历经历行业 |
| 5 | 职位级别匹配度 | 10% | JD 级别 vs 简历当前级别 |
| 6 | 薪资期望匹配度 | 5%（可选） | JD 薪资范围 vs 简历期望（若无薪资数据则不计分） |
| 7 | 地理位置匹配度 | 5% | JD 工作地点 vs 简历中候选人所在地 |
| 8 | 跳槽频率匹配度 | 5%（可选） | JD 对稳定性的隐性要求 vs 简历实际换工作频率 |

**评分标准（各维度）：**

- 优秀（90-100）：明显超出要求，或完全匹配
- 良好（70-89）：满足主要要求
- 一般（50-69）：部分满足要求，存在差距
- 较差（0-49）：明显不符合要求

**综合评分 = Σ（各维度得分 × 权重）**，最终输出 0-100 分。

**权重调优：** 系统内置不同行业/职位的默认权重配置，HR 可在职位级别调整；后续随面试通过率数据积累，AI 持续调优权重参数。

**F-12：阈值规则与候选人分流**

| 匹配分区间 | 系统行为 |
|---|---|
| ≥ 阈值（默认 80） | 进入"待通知"状态，HR 可发送面试邀请 |
| 60 - 阈值-1 | 进入"备选池"，HR 可手动捞起 |
| < 60 | 进入"不合适"状态（不通知） |

阈值由 HR 在职位发布时设定（0-100），可随时在职位详情中修改。

**F-13：匹配透明度（参考 HireVue AI Explainability）**

候选人和 HR 有权了解匹配评分背后的逻辑：

- 每份简历的匹配报告显示：各维度得分明细 + 扣分原因说明
- 候选人不可以看到详细权重配置，但可以看到"技能匹配：75分 — 原因：缺少 Vue3 经验"
- HR 可以看到完整的各维度得分和权重配置

### 3.5 候选人通知与面试邀请

**F-14：面试邀请发送**

| 通知方式 | 触发条件 |
|---|---|
| 邮件 | 始终发送（简历/平台上如有邮箱则发送） |
| 短信 | 有手机号则发送，无则跳过 |
| 两者都无 | HR 手动通知（从备选池手动操作） |

邮件/短信内容包含：
- 应聘职位名称
- 公司名称
- 面试邀请链接（含时效性令牌，24小时内有效）
- 面试预计时长
- 面试前准备提示

**F-15：候选人自主预约时间**

候选人点击邀请链接后：
- 查看面试介绍和 пример 问题（可选，让候选人提前了解面试形式）
- 选择合适的面试时间（系统给出可用时间窗口，如未来 48 小时内的多个时段）
- 确认后，系统将面试时间通知 HR

**F-16：HR 实时通知**

当候选人进入面试时：
- 系统立即推送通知给 HR（飞书消息 / 邮件）
- HR 可选择：实时观看面试过程 / 事后查看报告

### 3.6 AI 实时视频面试

**F-17：面试问题配置**

每个职位可配置以下问题：

| 问题类型 | 数量 | 说明 |
|---|---|---|
| HR 预设必问题 | 3-5 题 | HR 手动配置，如"请介绍一下你最近的项目" |
| AI 动态追问 | 不限 | AI 根据候选人回答自动生成追问（深度挖掘） |

**AI 追问机制（参考 HireVue）：**
- AI 在候选人回答完每个预设问题后，生成 0-3 个追问（取决于回答深度）
- 追问聚焦于：技能的深度验证 / 经历的真实性核查 / 行为面试（STAR 追问）
- 面试总时长上限可配置（默认 20-30 分钟）

**F-18：实时面试流程**

1. 候选人准时进入面试链接（浏览器即可，无需下载 App）
2. AI 开头问候，告知面试流程
3. AI 逐一提出预设问题，等待候选人回答
4. 候选人口头回答（实时语音转文字，AI 同步理解）
5. AI 追问（每次追问后等待回答）
6. 所有问题完成后，AI 结束语
7. 面试全程录制（音频 + 实时转写文本）

**F-19：面试候选人体验（参考 HireVue 95% 完成率设计）**

- **时长合理：** 默认 20 分钟，不超过 30 分钟
- **操作简单：** 点击链接即可开始，无需下载任何软件
- **技术门槛低：** 只需浏览器 + 麦克风，支持手机和电脑
- **面试环境提示：** 进入前提示"请在安静环境中面试"，但不做强制环境检测
- **随时可退出：** 候选人可随时终止面试（系统记录"候选人主动放弃"）

**F-20：候选人体验公平性（参考 HireVue 的一致性原则）**

- 所有候选人回答**相同的问题集**（预设问题相同，AI 追问根据回答内容有差异）
- AI 评估基于**统一的评分标准**，不受候选人背景、学校、品牌公司等因素影响
- 评估聚焦于：**技能掌握程度 / 逻辑思维 / 沟通表达 / 经验真实性**

### 3.7 结构化面试报告

**F-21：报告生成**

每场面试结束后，AI 自动生成结构化报告，生成时间 ≤ 5 分钟。

**报告结构：**

```
┌─────────────────────────────────────────────┐
│           TalentPilot 面试评估报告            │
├─────────────────────────────────────────────┤
│ 【候选人基础信息】                             │
│ 姓名 │ 学历 │ 工作年限 │ 应聘职位 │ 面试时间  │
├─────────────────────────────────────────────┤
│ 【JD 匹配度回顾】                            │
│ 技能 85 │ 年限 78 │ 学历 90 │ 行业 60 │ ... │
├─────────────────────────────────────────────┤
│ 【面试过程记录】                              │
│ 总时长：18分32秒 │ 问题数：4题 │ 追问数：7次  │
│ 回答完整度：92%（各问题回答长度分析）           │
├─────────────────────────────────────────────┤
```

## 5. AI 公平性与合规（行业研究启示）

### 5.1 AI 伦理设计原则（参考 HireVue AI Ethical Principles）

**原则 1：一致性（Consistency）**
- AI 对所有候选人使用相同的评估标准
- 不因候选人的学校、品牌公司、性别、年龄等因素产生评分偏差
- 参考 HireVue 实践：候选人回答相同问题，评估相同标准

**原则 2：可解释性（Explainability）**
- 每个匹配分数都有维度明细和原因说明
- 面试报告中的 AI 评语必须有具体文本支撑，不能是纯分数
- HR 可以看到"为什么这个候选人得了这个分数"

**原则 3：定期审计（Regular Auditing）**
- 定期（每季度）对 AI 模型进行偏见审计
- 关注维度：不因性别/年龄/学历/学校产生显著评分差异
- 审计结果记录在案（v1 可先记录日志，v2 接入第三方审计）

### 5.2 候选人数据合规

- 候选人简历数据仅用于本系统招聘流程
- 候选人面试视频/音频在报告生成后，可应候选人要求删除
- HR 不得将候选人数据导出到系统外部
- 系统遵循《个人信息保护法》要求，候选人有权要求查阅/删除自己的数据

### 5.3 透明度要求

- 系统明确告知候选人"本次面试由 AI 主导"
- 候选人在面试前已知晓面试将全程录制
- 候选人有权拒绝 AI 面试，选择人工面试（届时系统降级为记录模式）

- 候选人数据不可导出到系统外部

## 8. MVP 优先级定义

考虑到开发资源，v1.0 MVP 仅实现核心链路，砍掉扩展功能。

### 8.1 v1.0 MVP 必须有（MVP Scope）

| 模块 | 功能 |
|---|---|
| 职位管理 | 创建/编辑/发布职位（仅支持手动发布，暂不定时） |

| AI 面试 | 单职位 3 个预设问题 + AI 追问 + 实时对话 |
| 面试报告 | 自动生成 + 在线查看（不含导出） |
| 权限 | Admin + HR 两角色 |

- 跨渠道自动发布和采集（Browser 自动化）
- 面试实时监控
- 报告导出（PDF/Excel）
- 用人经理角色
- 移动端适配
- AI 权重自动调优
